Contribute to Eta

Eta and the associated tools and libraries are all BSD 3-clause licensed open-source projects that require contributors to sign the TypeLead CLA. The majority of the Eta team works at TypeLead which takes an active role in pushing the ecosystem forward, but there have been many external contributors who have helped make Eta awesome. When you have something to contribute, but aren't sure about whether it will be useful to the community, you are free to join any of the active communication channels to discuss with the community. Even if you are new to functional programming, there are several ways in which you can help shown below.

Documentation

Great documentation is essential to help newcomers and experts alike and you can never have enough of it! Many aspects of Eta’s documentation don’t require deep knowledge to improve. Help out by writing, reviewing, and editing documentation, and gain a deeper understanding of Eta in the process. Writing helps solidify concepts that may have been vague before. Don’t like the way something reads? Discover some information that wasn’t documented? Send in a pull request!

Eta & Etlas User Guides

Overview

The user guides provide a detailed look at all the features that are available and describe them in detail so that developers can look them up when the need arises. User guides can always use more information and simpler exposition.

Guidelines

Keep the wording simple and to the point. Be sure to explain all the details about the feature in question.

The Boot Libraries

Overview

The standard libraries, also known as the boot libraries are the core part of the Eta language. In fact, they are contain the most frequently used functions in practice so it is important to keep evolving them to make sure they keep up with the ecosystem.


The boot libraries include the following:

  • ghc-prim: The home to all the primitive operations and definitions for the standard data types and type classes. The source code can be found in libraries/ghc-prim in the main Eta repo.
  • integer: Contains the implementation of the arbitrary-precision Integer type. The source code can be found in libraries/integer in the main Eta repo.
  • base: Contains the Prelude module which is included into every Eta module by default, along with many everyday functions like map. The source code can be found in libraries/base in the main Eta repo.

Most documentation pull requests to the standard libraries are welcome so you don't need to consult the community first.

Guidelines

The documentation for the boot libraries is probably the most frequently read so take extra care to make sure the documentation you write is simple and easy to understand.

Runtime System

Overview

The runtime system is written in Java and the documentation is present in the source code in the form of Javadoc annotations.


Better documentation for the runtime system means that it will be easier for newcomers to contribute and help develop useful extensions.


The source code for the runtime system can be found at rts in the main Eta repo.


Guidelines

Be very precise with the details written in the documentation and don't hesitate to ask the Eta team if you are not sure about the correctness of your documentation.

Compiler

Overview

The compiler is completely written in Haskell and documentation for the compiler is in the source code in the form of documentation comments that are parsed by the Haddock documentation tool.


Better documentation for the compiler means that it will be easier for newcomers to contribute and help the compiler and language evolve faster.


The source code for the compiler can be found at compiler in the main Eta repo.


The documentation annotations in the source code have a specific format which you can read about in the Haddock user guide.

Guidelines

Be very precise with the details written in the documentation and don't hesitate to ask the Eta team if you are not sure about the correctness of your documentation.

Standard Library

Overview

The Eta standard library is very similar to GHC's standard library with additional reimplementation required for JVM support. We are still working on the complete documentation of the standard library. But you can check the comments in the standard library for now.

Get started

If you want to get started, check for [standard library] tag in the Eta repo.

Libraries & Packages

Overview

Since, Eta is a new language there is significant room for new libraries. We have already ported a great deal of Hackage packages to Eta which can be found here. We are also working on a package repository called 'Etlas' for Eta.

Get started

You can either port a Haskell or Java package to Eta. And let us know so that we can add your library to our main repo.
